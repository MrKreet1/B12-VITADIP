# Скрининг геометрии B12 в ORCA 6.1

Набор скриптов для поиска наиболее стабильной геометрии кластера
из 12 атомов бора путём скрининга множества стартовых конфигураций.

Итоговый отчет по проведенным расчетам: [`REPORT.md`](REPORT.md).  
Академическая версия отчета: [`ACADEMIC_REPORT.md`](ACADEMIC_REPORT.md).

## Что в комплекте

| Файл                | Назначение                                               |
|---------------------|----------------------------------------------------------|
| `generate_configs.py` | Генерирует 70+ стартовых XYZ-файлов в `./configs/`     |
| `make_inputs.py`    | Превращает XYZ в ORCA input-файлы в `./jobs/<name>/`     |
| `run_batch.sh`      | Последовательно запускает ORCA на всех заданиях          |
| `analyze_results.py`| Парсит .out, строит таблицу энергий, находит минимум     |
| `refine_top.py`     | Готовит пересчёт топ-N на более точном уровне теории     |

## Метод расчёта (по умолчанию)

**Скрининг (быстро):** BP86 / def2-SVP / RI-J / D3BJ / Opt / TightSCF.
Одна оптимизация B12 ≈ 10–30 минут на 8 ядрах.

**Уточнение топ-5:** B3LYP / def2-TZVP / RIJCOSX / D3BJ / Opt + Freq.
Freq нужен, чтобы отсеять седловые точки (все частоты должны быть > 0).

## Требования

- ORCA 6.1, полный путь должен быть в `PATH` (`which orca` должен работать).
- Python 3.8+, `numpy`.
- Конфигурация параллелизма: внутри ORCA через `%pal nprocs 8`.
  Скрипт `run_batch.sh` запускает задачи **последовательно** — каждая
  получает все 8 ядер. Одновременный запуск нескольких ORCA на 8 ядрах
  не даст ускорения.
- Память: `%maxcore 2500` МБ на ядро × 8 = 20 ГБ, оставит запас под ОС.

## Пошагово

### 1. Сгенерировать стартовые конфигурации

```bash
python3 generate_configs.py
```

Создаст `./configs/*.xyz` — 78 конфигураций:
- 7 топологий (икосаэдр, кубооктаэдр, гекс. призма/антипризма,
  усечённый тетраэдр, квазипланарная «чаша», двойное кольцо)
  × 6 масштабов (nn = 1.6, 1.8, 2.0, 2.5, 3.0, 5.0 Å);
- возмущённые копии 4 топологий (по 3 сида × 2 масштаба);
- 6 случайных конфигураций × 2 масштаба.

**Замечание про 5 Å.** Равновесное B–B ~1.6–1.8 Å. При старте с 5 Å
атомы практически не связаны; такие расчёты либо развалятся, либо
дадут очень высокую энергию. 5 Å оставлен как контрольная точка.

### 2. Сгенерировать ORCA-входы (скрининг, синглет)

```bash
python3 make_inputs.py
```

Создаст `./jobs/<name>/<name>.inp`. Для триплета:

```bash
python3 make_inputs.py --mult 3 --suffix _triplet
```

### 3. Запустить расчёты

```bash
chmod +x run_batch.sh
./run_batch.sh jobs
```

Скрипт пропускает уже успешно завершённые задания (`ORCA TERMINATED
NORMALLY`), поэтому можно прерывать и продолжать.
Прогресс: `jobs/_progress.log`.

Для фонового запуска (лог + nohup):

```bash
nohup ./run_batch.sh jobs > batch.log 2>&1 &
tail -f batch.log
```

Оценка времени: 78 задач × ~20 мин ≈ 25 часов. Ставьте на ночь/сутки.

### 4. Анализ результатов

```bash
python3 analyze_results.py
```

На выходе:
- `results.csv` — таблица по всем задачам (ранг, статус, E, ΔE);
- `all_optimized.xyz` — мульти-XYZ, отсортированный по энергии
  (открывается в VMD, Avogadro, OVITO);
- `best_structure.xyz` — структура с минимальной энергией;
- сводка в консоль с топ-10.

### 5. Уточнение топ-кандидатов (рекомендуется)

BP86/def2-SVP в скрининге склонен путать порядок близких по энергии
изомеров. После скрининга:

```bash
python3 refine_top.py --top 5
./run_batch.sh jobs_refined
python3 analyze_results.py --jobs-dir jobs_refined \
        --out-csv results_refined.csv \
        --out-combined all_refined.xyz \
        --out-best best_refined.xyz
```

`Freq` в refine-входе даст частоты. В `<name>.out` ищите
`The first frequency`; если встречается мнимая частота
(`imaginary mode`), это седловая точка — исключайте её из кандидатов
в глобальный минимум.

### 6. Триплет (опционально, но полезно)

Для B12 в нейтральном состоянии и синглет, и триплет — разумные
варианты основного состояния для разных изомеров. После refine
синглетов запустите те же топ-5 в триплете:

```bash
python3 make_inputs.py --mult 3 --suffix _triplet_refine \
        --method B3LYP --basis def2-TZVP --ri RIJCOSX \
        --configs-dir configs   # или другая директория с XYZ
./run_batch.sh jobs_triplet_refine
```

## Кастомизация

Все параметры метода — флаги `make_inputs.py`:

```bash
python3 make_inputs.py \
    --method PBE0 --basis def2-TZVP --ri RIJCOSX \
    --charge 0 --mult 1 --nprocs 8 --maxcore 2500
```

Изменить список масштабов nn или набор топологий — редактируйте
`nn_scales` и `topologies` в `generate_configs.py`.

## Типовые проблемы

- **`orca: command not found`** в `run_batch.sh`: добавьте ORCA в PATH,
  например `export PATH=/opt/orca_6_1_0:$PATH`.
- **SCF not converged**: попробуйте `! SlowConv` (уже включён) и
  `! SOSCF`; для плохих стартовых геометрий — `! Opt LooseOpt` на
  первом шаге, потом дооптимизировать `! TightOpt`.
- **Геометрия разошлась** (5 Å старт): ожидаемо. Смотрите `results.csv`,
  status = `NOT_CONVERGED` или `CRASHED` — такие просто исключайте.
- **RAM**: если видите OOM, уменьшите `--maxcore` до 2000.
- **MPI / OpenMPI**: ORCA 6.1 требует корректную OpenMPI в PATH/LD_LIBRARY_PATH,
  ту, с которой она собрана. Проверьте документацию вашей сборки.

## Ожидаемый результат для B12

По опубликованным данным глобальный минимум нейтрального B12 —
**квазипланарная выпуклая структура** (мотив `quasi_planar_convex`,
симметрия близка к C3v). Икосаэдр B12 устойчив только в составе
B12H12²⁻ и производных; в голом B12 он оказывается существенно
выше по энергии. Скрининг должен это воспроизвести: лучшая структура
после refine, как правило, именно квазипланарная.
